Clicksign API 3.0 - Complete Research Notes
=============================================

Base URL: 
  Production: https://app.clicksign.com/api/v3
  Sandbox: https://sandbox.clicksign.com/api/v3

Authentication: 
  Header: Authorization: Bearer {access_token}
  Content-Type: application/vnd.api+json

=== KEY ENDPOINTS ===

1. LIST ENVELOPES (contracts)
   GET /envelopes
   Query params: filter[status]=running|closed|canceled|draft, filter[name], filter[created], sort=name|-name
   Returns: paginated list of envelopes

2. ENVELOPE DETAILS
   GET /envelopes/{envelope_id}
   Returns: envelope with name, status, created_at, updated_at

3. LIST SIGNERS (per envelope)
   GET /envelopes/{envelope_id}/signers
   Returns: list of signers with:
     - name (string)
     - email (string)
     - phone_number (string, 10-11 digits)
     - documentation (string, CPF format: 123.321.123-40)
     - has_documentation (boolean)
     - birthday (date, YYYY-MM-DD)

4. LIST DOCUMENTS (per envelope)
   GET /envelopes/{envelope_id}/documents
   Returns: list of documents

5. DOCUMENT DETAILS
   GET /envelopes/{envelope_id}/documents/{document_id}
   Returns: document with filename, status, metadata
   Key fields:
     - filename (string)
     - content_base64 (if uploaded)
     - template (object, if from template - contains template key + data)
     - metadata (object, free-form - can contain VIN!)

=== DATA EXTRACTION STRATEGY ===

From SIGNERS → CPF (documentation), Name, Email, Phone
From DOCUMENTS → VIN extraction:
  Option A: template.data may contain VIN as a variable
  Option B: metadata may contain VIN
  Option C: filename may contain VIN
  Option D: Parse document content (content_base64 → decode → search for VIN pattern)

VIN Regex Pattern: /\b[A-HJ-NPR-Z0-9]{17}\b/g (standard 17-char)
Also support shorter Brazilian chassis: /\b[A-Z0-9]{6,17}\b/g

=== INTEGRATION FLOW ===

1. Request CLICKSIGN_ACCESS_TOKEN secret from user
2. Fetch all envelopes (paginated, filter[status]=closed for signed contracts)
3. For each envelope:
   a. Get signers → extract CPF, name, email, phone
   b. Get documents → check metadata/template for VIN, or parse content
   c. Match signer CPF to VIN (1 signer = 1 contract = 1+ VINs)
4. Upsert customer by CPF
5. For each VIN:
   a. Check if VIN exists in vehicles table
   b. If VIN exists in bl_vehicles → status = em_processo (auto)
   c. If VIN not in bl_vehicles → status = aguardando_embarque (default)
6. Store clicksign_envelope_id and clicksign_signer_id for traceability

=== CUSTOMER STATUS ENUM ===
- aguardando_embarque: Contract signed, awaiting shipping start
- aguardando_li: Import to Brazil, awaiting Import License
- em_processo: VIN found in existing BL (automatic)
- concluido: Vehicle delivered
- cancelado: Contract canceled
