ARCHITECTURAL PROBLEM:
- The current schema has bills_of_lading.vehicle_id as a SINGLE FK → vehicles
- But a BL can contain MULTIPLE vehicles (containers with multiple cars)
- The current workaround is to put all VINs in vehicle_description as text
- Only single-vehicle BLs get a proper vehicle_id link

- The vehicles table has customer_id (FK → customers) — this is correct
- But customers table is EMPTY — no customers have been created yet

RELATIONSHIP CHAIN (as designed):
  Customer (CPF) → Vehicle (VIN) → BL (bl_number) → Tracking Events
  Customer (CPF) → Tracking Code → BL → Tracking Events

WHAT'S MISSING:
1. A many-to-many relationship between BLs and Vehicles (bl_vehicles junction table)
2. Customers are not being created/linked during agent ingestion
3. The VIN is the bridge between customer and BL, but currently:
   - BL has vehicle_id (single FK, doesn't support multi-vehicle BLs)
   - Vehicle has customer_id (correct, but no customers exist)
   - Tracking code has both bl_id and customer_id (correct design)
