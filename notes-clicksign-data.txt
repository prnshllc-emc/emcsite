Clicksign API v1 Data Analysis
================================

API: GET /api/v1/documents?access_token=...&page=1&per_page=10
Returns: { documents: [...], page_info: { total_pages, current_page, next_page } }

Document list keys: key, folder_id, filename, uploaded_at, updated_at, finished_at, deadline_at, status, auto_close
Document detail keys: key, account_key, path, filename, uploaded_at, updated_at, finished_at, deadline_at, status, auto_close, locale, metadata, sequence_enabled, signable_group, remind_interval, block_after_refusal, preview, downloads, template, signers, events, attachments, links

Signer keys: key, name, email, phone_number, documentation (CPF), has_documentation, birthday, created_at

IMPORTANT FINDINGS:
1. The list endpoint does NOT include signers — need to call detail endpoint per document
2. Signers include: name, email, phone_number, documentation (CPF format: 337.381.238-06)
3. Some signers have has_documentation=false and null documentation (no CPF)
4. One signer is always Frederico Junqueira (company owner) — need to filter him out
5. The other signer is the CLIENT
6. VIN is NOT in structured fields — must parse from filename or download content
7. Filename pattern: "CONTRATO IMPORTACAO VEICULOS BRASIL [CLIENT NAME].pdf" or "CAR SHIPPING CONTRACT [CLIENT NAME].pdf"
8. Status: "closed" = signed, "canceled" = canceled
9. template field is null for uploaded PDFs (not template-based)
10. content_base64 is NOT returned in detail — need to use downloads.original_file_url
